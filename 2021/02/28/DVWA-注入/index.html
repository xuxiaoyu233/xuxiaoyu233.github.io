第一部分(SQL Injection)

SQL Injection，即SQL注入，是指攻击者通过注入恶意的SQL命令，破坏SQL查询语句的结构，从而达到执行恶意SQL语句的目的。SQL注入漏洞的危害是巨大的，常常会导致整个数据库被“脱裤”，尽管如此，SQL注入仍是现在最常见的Web漏洞之一。

第二部分（SQL Injection(Blind)）

SQL Injection（Blind），即SQL盲注，与一般注入的区别在于，一般的注入攻击者可以直接从页面上看到注入语句的执行结果，而盲注时攻击者通常是无法从显示页面上获取执行结果，甚至连注入语句是否执行都无从得知，因此盲注的难度要比一般注入高。目前网络上现存的SQL注入漏洞大多是SQL盲注。

Sqlmap

我们直接进入正题，了解一下Sqlmap对post型和get型使用时的区别。

GET型

对于get型，sqlmap较为直接，直接操作（相对来说直接），但有时候也时需要一些信息辅助的。

比如想要在DVWA内进行sqlmap注入，拿low级别的盲注来说吧，直接进行sqlmap -u “注入点” 这样的测试，返回的一定是错误。这是因为对于登录型的网页来说我们必须要有身份的验证和一定的权限，所以我们需要配合burp suite工具抓包得到cookie再进行注入。
